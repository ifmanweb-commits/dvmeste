## 📋 ТЗ: Раздел сообщений с модераторами в личном кабинете

### 1️⃣ Общая информация

**Проект:** Давай вместе — каталог психологов
**Технологии:** Next.js 16 (App Router), Prisma, Tailwind CSS
**Тип задачи:** Создание системы обмена сообщениями между психологами и модераторами

### 2️⃣ Структура базы данных

```prisma
model Message {
  id          String   @id @default(cuid())
  userId      String   // ID психолога (чей это диалог)
  direction   String   // 'to_moder' или 'to_user'
  text        String   @db.Text
  isRead      Boolean  @default(false)
  createdAt   DateTime @default(now())

  user        User     @relation(fields: [userId], references: [id])

  @@index([userId])
  @@index([createdAt])
}
```

### 3️⃣ Функциональные требования

#### А. Страница сообщений в ЛК
**URL:** `/account/messages`

**Компоненты страницы:**
- Заголовок "Сообщения модераторам"
- Форма отправки нового сообщения
- Лента сообщений с пагинацией

#### Б. Форма отправки
- Только textarea (многострочное поле ввода)
- Кнопка "Отправить"
- **Валидация:**
  - Текст не может быть пустым
  - Очистка от HTML-тегов (предотвращение XSS)
  - Сохранение переносов строк (pre-wrap)
- После отправки:
  - Поле очищается
  - Новое сообщение появляется в ленте (без перезагрузки)
  - Лента прокручивается вверх к новому сообщению

#### В. Лента сообщений
- **Пагинация:** 20 сообщений за раз
- **Кнопка "Загрузить ещё"** внизу ленты
- **Сортировка:** новые сверху
- **Отображение:**
  - Сообщения пользователя — справа (выравнивание по правому краю)
  - Сообщения модератора — слева (выравнивание по левому краю)
  - Визуальное отличие (разные фоны)
  - Дата и время отправки (формат: ДД.ММ.ГГГГ ЧЧ:ММ)
  - Текст с сохранением переносов (pre-wrap)
  - Кликабельные ссылки (превращаются в активные)

#### Г. Логика isRead
- Когда пользователь загружает страницу `/account/messages`:
  - Все сообщения, где `toUserId = текущий пользователь`, помечаются `isRead = true`
- Это нужно для будущей аналитики и счетчиков (сейчас не показываем, но данные собираем)

### 4️⃣ Правила отправки

**Кто может отправлять:**
- Психологи с любым статусом (кроме BLOCKED - их не пускать в раздел, отображать заглушку, не отображать в меню.)
- Модераторы и админы (через админку, пока не делаем)

**Куда отправляются сообщения от психолога:**
- Все сообщения психолога попадают в **общий ящик модераторов**
- Сообщения от пользователя имеют direction='to_moder' и наоборот.

### 5️⃣ Безопасность

- Очистка HTML из сообщений (чтобы нельзя было вставить скрипты)
- Защита от XSS при отображении
- Проверка прав доступа:
  - Пользователь видит только свои сообщения
  - При запросе сообщений проверять `fromUserId` или `toUserId`

### 7️⃣ Технические требования

#### Server Actions (в `lib/actions/messages.ts`)

```typescript
// Отправить сообщение
export async function sendMessage(text: string)

// Получить сообщения (с пагинацией)
export async function getMessages(page: number = 1, limit: number = 20)

// Отметить сообщения как прочитанные (вызывается при загрузке страницы)
export async function markMessagesAsRead()
```

#### Страница в ЛК
`app/account/messages/page.tsx` — клиентский компонент

#### Компоненты
- `MessagesClient` — основной компонент
- `MessageForm` — форма отправки
- `MessageList` — список сообщений с пагинацией
- `MessageBubble` — отдельное сообщение

### 8️⃣ Дополнительные требования

- **Обработка ссылок:** превращать URL в тексте в кликабельные `<a href="...">`
- **pre-wrap:** все сообщения отображать с CSS `white-space: pre-wrap`
- **Экранирование:** перед отправкой очищать текст от HTML-тегов (оставлять только текст и переносы)
- **Ошибки:** показывать понятные сообщения при ошибках отправки
- **Загрузка:** индикатор загрузки при отправке и подгрузке сообщений

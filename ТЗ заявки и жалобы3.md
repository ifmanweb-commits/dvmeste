## 📋 ТЗ: Этап 3 — Личный кабинет психолога: раздел "Заявки"

### 1️⃣ Общая информация

**Проект:** Давай вместе — каталог психологов  
**Технологии:** Next.js 16 (App Router), Tailwind CSS  
**Тип задачи:** Создание страницы списка заявок для психолога и детальной страницы заявки

---

### 2️⃣ Что нужно сделать

#### 2.1 **Страница списка заявок** (`/account/leads`)

**Вкладки:**
- **Новые** — статус `NEW`
- **Принятые** — статусы `ACCEPTED`, `CONTACTED`, `APPOINTMENT`, `FREE_SESSION`, `PAID_SESSION`
- **Архив** — статусы `REJECTED`, `NO_CONTACT`, `CLIENT_REJECTED`, `ARCHIVED`

**Карточка заявки:**
- Дата и время создания
- Имя клиента (если есть)
- Первые 70 символов сообщения
- Статус (бейдж)
- Если `isSuspicious = true` — плашка с предупреждением:  
  ⚠️ "На этого клиента жаловались N раз. Будьте осторожны"
- Кнопка "Подробнее" → ведёт на `/account/leads/[id]`

**Пагинация:** 20 заявок на страницу, кнопка "Загрузить ещё" или "Предыдущая/Следующая"

---

#### 2.2 **Детальная страница заявки** (`/account/leads/[id]`)

**Отображаемая информация:**
- Все данные клиента: имя, email, телефон, Telegram/VK
- Полное сообщение клиента
- Статус (бейдж)
- Дата создания
- Дата изменения статуса
- Если `isSuspicious = true` — плашка с предупреждением

**Кнопки действий (зависят от статуса):**

| Текущий статус | Доступные кнопки |
|----------------|------------------|
| `NEW` | ✅ Принять / ❌ Отказаться |
| `ACCEPTED` | 📞 Связался / 📅 Договорились / 👎 Клиент отказался / 📦 В архив |
| `CONTACTED` / `APPOINTMENT` | 💰 Бесплатная сессия / 💵 Платная сессия / 👎 Клиент отказался / 📦 В архив |
| `FREE_SESSION` / `PAID_SESSION` | 📦 В архив |

**Отказаться (`REJECTED`):**
- Модалка с полями:
  - Причина для клиента (обязательно, отправится на email)
  - Причина для себя (опционально, не показывается клиенту)

**Пожаловаться на клиента:**
- Кнопка ⚠️ "Пожаловаться" внизу страницы
- Модалка с полями:
  - Причина (выпадающий список: спам, хамство, неадекватное поведение, другое)
  - Описание (текстовое поле, обязательно)

### 5️⃣ Требования к интерфейсу

- **Страница списка** — заголовок "Заявки", вкладки, карточки
- **Карточка** — белый фон, граница, отступы, скругление
- **Бейдж статуса** — цвет в зависимости от статуса
- **Плашка предупреждения** — жёлтый фон, значок ⚠️
- **Кнопки** — цветные (зелёные для успешных действий, красные для опасных, серые для нейтральных)
- **Модалки** — затемнённый фон, по центру, с заголовком, полями и кнопками

### 6️⃣ Взаимодействие с API

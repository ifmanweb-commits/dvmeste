---

### 2️⃣ Что нужно сделать

#### 2.1 **Компонент формы заявки**

Создать компонент `LeadFormModal.tsx` в папке `components/lead/`:

**Функциональность:**
- Модальное окно, которое открывается по кнопке "Связаться"
- Поля:
  - Имя (текстовое поле, опционально)
  - Email (email, обязательно)
  - Телефон (tel, опционально)
  - Telegram/VK/другие контакты (текстовое поле, опционально)
  - Сообщение (textarea, обязательно)
  - Чекбокс "Я согласен на обработку персональных данных" (обязательный)
  - Чекбокс "Запомнить меня для будущих заявок" (опциональный)

**Логика:**
1. При открытии формы проверить куку `clientId` и подтянуть сохранённые данные (имя, email, телефон)
2. Валидация на клиенте (проверка email, непустого сообщения, согласия)
3. Отправка через API `/api/leads` (или Server Action `createLead`)
4. Индикатор загрузки при отправке
5. Показ сообщения об успехе/ошибке
6. Закрытие модалки после успешной отправки

#### 2.2 **Кнопка "Связаться" на странице психолога**

На странице `/catalog/[slug]` добавить кнопку "Связаться" (рядом с "Записаться").

При клике — открывается модалка.

#### 2.3 **Скрытая защита от ботов**

Добавить в форму:
- `honeypot` — скрытое поле `website`, заполняют только боты
- Проверка времени заполнения (на сервере) — если форма отправлена быстрее 3 секунд — отклонять

### 5️⃣ Требования к интерфейсу

- **Модалка** — на всю ширину на мобильных, 500px на десктопе
- **Кнопка** "Связаться" — стилизована как `bg-[#5858E2] text-white`
- **Валидация** — красные сообщения под полями
- **Индикатор загрузки** — спиннер на кнопке отправки
- **Сообщение об успехе** — зелёная плашка внутри модалки

---

### 6️⃣ Взаимодействие с API

Использовать готовый API роут `/api/leads`:

```typescript
const response = await fetch('/api/leads', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    psychologistId,
    client: {
      email,
      name,
      phone,
      telegram
    },
    message,
    consent: true,
    rememberMe: true/false
  })
})
```

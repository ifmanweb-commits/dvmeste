## 📋 ТЗ: Раздел модерации фотографий психологов

### 1️⃣ Общая информация

**Проект:** Давай вместе — каталог психологов
**Технологии:** Next.js 16 (App Router), Prisma, Tailwind CSS
**Тип задачи:** Создание раздела модерации фотографий профилей психологов

### 2️⃣ Текущее состояние

#### Модель Document (существующая)
```prisma
model Document {
  id           String       @id @default(cuid())
  type         DocumentType // 'PHOTO'
  url          String       // путь к файлу
  filename     String
  userId       String
  verifiedAt   DateTime?    // null = не проверено
  verifiedBy   String?      // кто проверил
  uploadedAt   DateTime     @default(now())
  user         User         @relation("user_documents", fields: [userId], references: [id])
}

enum DocumentType {
  PHOTO
  ACADEMIC_EDUCATION
  PROFESSIONAL_TRAINING
  COURSE
  // ... остальные
}
```

#### Модель User (существующая)
```prisma
model User {
  avatarUrl String? // ссылка на главное фото
  // ... остальные поля
}
```

### 3️⃣ Функциональные требования

#### А. Страница модерации фото
**URL:** `/admin/moderation/photos`

**Что должно быть:**
- Список всех фотографий со статусом `verifiedAt: null`
- **Группировка по пользователям** (психологам)
- Сортировка: сначала новые загрузки

#### Б. Интерфейс для каждого пользователя

```
Иванов Иван (ivan@mail.com)
┌────────┐ ┌────────┐ ┌────────┐
│   Фото │ │   Фото │ │   Фото │
│   1/3  │ │   2/3  │ │   3/3  │
│        │ │        │ │        │
│ [✅][❌]│ │ [✅][❌]│ │ [✅][❌]│
└────────┘ └────────┘ └────────┘
[✅ Принять все]
```

**Элементы:**
- ФИО и email психолога (ссылка на профиль в админке)
- Миниатюры фотографий (превью)
- Под каждой фото:
  - Кнопка "Принять" (зеленая)
  - Кнопка "Отклонить" (красная)
- Кнопка "Принять все" под группой фото пользователя

#### В. Логика действий

**Принять фото:**
- Устанавливает `verifiedAt = new Date()`
- Фото становится видимым в профиле
- Ничего не удаляется

**Отклонить фото:**
1. Проверить, является ли это фото аватаром (`avatarUrl === photo.url`)
2. Если да — сбросить `avatarUrl: null` у пользователя
3. Удалить файл с диска (`fs.unlink`)
4. Удалить запись из таблицы Document

**Принять все фото пользователя:**
- Установить `verifiedAt = new Date()` для всех непроверенных фото этого пользователя

#### Г. Пустые состояния
- Если нет фото на модерацию — показывать заглушку
- Если у пользователя нет фото — не показывать в списке

### 4️⃣ Технические требования

#### Server Actions (в `lib/actions/moderation-photos.ts`)


#### Компоненты

### 5️⃣ Визуальные требования

**Цвета:**
- Синий `#5858E2` — основной цвет
- Зеленый — кнопки "Принять"
- Красный — кнопки "Отклонить"

## 📁 Базовый путь для файлов

В проекте используется единая структура хранения файлов в папке **`/public/files/`** с подпапками по типу сущности:

```
/public/files/
├── articles/           # файлы статей
│   └── [articleId]/
├── pages/              # файлы для страниц CMS
│   └── [pageId]/
├── users/              # файлы пользователей (фото, документы)
│   └── [userId]/
│       ├── photo/      # фотографии профиля
│       └── doc/        # документы (дипломы, сертификаты)
```

Для фотографий пользователей путь формируется так:
```
/public/files/users/[userId]/photo/[filename].webp
```

**Полный URL:** `/files/users/[userId]/photo/[filename].webp`

## 📦 Библиотека для работы с файлами

В проекте используется **кастомный API роут `/api/files`**, который:
- Загружает файлы
- Удаляет файлы
- Возвращает список файлов

**Основные файлы:**
- `app/api/files/route.ts` — API эндпоинты
- В ЛК для фото используется кастомная логика через `fetch` к `/api/files`

### Пример использования при удалении:

```typescript
import { promises as fs } from "fs";
import path from "path";

// Путь к файлу
const filePath = path.join(process.cwd(), "public", photo.url);

// Удаление
await fs.unlink(filePath).catch(() => {
  console.log("Файл уже удален или не существует");
});
```

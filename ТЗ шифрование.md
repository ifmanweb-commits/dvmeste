## 📋 ТЗ: Переход на поиск по emailHash

### 1️⃣ **Общая задача**
Заменить все прямые поиски по полю `email` на поиск по полю `emailHash` во всём проекте.

### 2️⃣ **Почему**
- Поле `email` теперь зашифровано (AES-256)
- Поиск по зашифрованному полю невозможен (недетерминированное шифрование)
- Для поиска нужно использовать хеш (SHA-256) — поле `emailHash`

### 3️⃣ **Где искать**

#### A. Авторизация (Magic Link)
- `lib/auth/magic-link.ts` — поиск пользователя по email при отправке ссылки
- `app/api/auth/verify/route.ts` — верификация токена

#### B. Админка
- Поиск психологов по email в админ-панели
- `lib/actions/admin-psychologists.ts` — функции `getPsychologistsList`, `getCandidatesList` и другие, где есть поиск по email

#### C. API роуты
- `app/api/auth/verify/route.ts` — поиск пользователя
- Другие API, где ищут пользователя по email

#### D. Server Actions
- Любые `prisma.user.findUnique({ where: { email } })`

#### E. Утилиты
- `lib/auth/session.ts` — получение пользователя по email (если есть)
- Функции, связанные с уведомлениями

### 4️⃣ **Как переделать**

**Было:**
```typescript
const user = await prisma.user.findUnique({
  where: { email: inputEmail }
})
```

**Стало:**
```typescript
import { createHash } from 'crypto'

const emailHash = createHash('sha256').update(inputEmail).digest('hex')

const user = await prisma.user.findUnique({
  where: { emailHash }
})
```

### 5️⃣ **Важно**
- Не менять чтение/отображение email — они автоматически расшифровываются плагином
- Менять только **поиск** (`where: { email }`)
- Добавить `import { createHash } from 'crypto'` в файлы, где используется поиск

### 6️⃣ **Проверка**
- После изменений авторизация должна работать
- Поиск психологов в админке по email — должен работать
- Создание новых пользователей — автоматически шифрует email и заполняет хеш

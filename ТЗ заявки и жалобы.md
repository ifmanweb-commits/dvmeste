### 2️⃣ Модели данных (добавить в `schema.prisma`)

```prisma
model Client {
  id              String   @id @default(cuid())
  email           String   @unique
  name            String?
  phone           String?
  telegram        String?
  vk              String?
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt

  // Репутация (невидимая для клиента)
  complaintCount  Int      @default(0)
  isShadowBanned  Boolean  @default(false)

  leads           Lead[]
  complaintsFrom  Complaint[] @relation("client_complaints_from")
  complaintsTo    Complaint[] @relation("client_complaints_to")

  @@index([email])
}

model Lead {
  id               String     @id @default(cuid())
  clientId         String
  psychologistId   String
  message          String?    @db.Text
  status           LeadStatus @default(NEW)
  statusChangedAt  DateTime?
  createdAt        DateTime   @default(now())
  viewedAt         DateTime?

  // Для подозрительных клиентов
  isSuspicious     Boolean    @default(false)
  suspiciousReason String?

  client           Client     @relation(fields: [clientId], references: [id], onDelete: Cascade)
  psychologist     User       @relation(fields: [psychologistId], references: [id], onDelete: Cascade)

  @@index([clientId])
  @@index([psychologistId])
  @@index([status])
  @@index([createdAt])
}

model Complaint {
  id                 String   @id @default(cuid())
  fromType           String   // 'client' или 'psychologist'
  fromClientId       String?
  fromPsychologistId String?
  toClientId         String?
  toPsychologistId   String?
  reason             String
  description        String?  @db.Text
  leadId             String?
  createdAt          DateTime @default(now())
  resolvedAt         DateTime?
  resolvedBy         String?
  resolution         String?

  fromClient         Client?  @relation("client_complaints_from", fields: [fromClientId], references: [id])
  fromPsychologist   User?    @relation("psychologist_complaints_from", fields: [fromPsychologistId], references: [id])
  toClient           Client?  @relation("client_complaints_to", fields: [toClientId], references: [id])
  toPsychologist     User?    @relation("psychologist_complaints_to", fields: [toPsychologistId], references: [id])
  lead               Lead?    @relation(fields: [leadId], references: [id])

  @@index([fromType])
  @@index([toClientId])
  @@index([toPsychologistId])
}

enum LeadStatus {
  NEW
  ACCEPTED
  REJECTED
  CONTACTED
  APPOINTMENT
  FREE_SESSION
  PAID_SESSION
  NO_CONTACT
  CLIENT_REJECTED
  ARCHIVED
}
```

### 3️⃣ Server Actions (`lib/actions/leads.ts`)

#### 3.1 `createLead(data: CreateLeadInput)`

**Входные данные:**
```typescript
interface CreateLeadInput {
  psychologistId: string
  client: {
    email: string
    name?: string
    phone?: string
    telegram?: string
    vk?: string
  }
  message: string
  rememberMe?: boolean  // сохранить clientId в cookies
  consent: boolean      // согласие на обработку ПД
}
```

**Логика:**
1. Проверка `consent` — если false, вернуть ошибку
2. Проверка лимита заявок (не больше 5 за последний час с этого email)
3. Поиск клиента по email (через `emailHash` — если шифрование готово)
4. Если клиент не найден — создать
5. Проверить `isShadowBanned` и `complaintCount` для установки `isSuspicious`
6. Создать заявку со статусом `NEW`
7. Если `rememberMe` — установить cookie `clientId`
8. Отправить уведомления психологу:
   - Email (через существующий nodemailer/unisender)
   - Push (если подписан)
9. Вернуть `{ success: true, leadId: string }`

#### 3.2 `getPsychologistLeads(psychologistId: string, filters: LeadFilters)`

**Входные данные:**
```typescript
interface LeadFilters {
  status?: LeadStatus
  page?: number
  limit?: number
}
```

**Логика:**
- Вернуть заявки психолога с пагинацией
- Включить данные клиента (имя, email, телефон)
- Для каждой заявки добавить флаг `isSuspicious` и `suspiciousReason`
- Сортировка: сначала новые

#### 3.3 `getLeadById(leadId: string, psychologistId: string)`

**Логика:**
- Проверить, что заявка принадлежит психологу
- Вернуть заявку с полными данными клиента

#### 3.4 `updateLeadStatus(leadId: string, status: LeadStatus, data?: UpdateLeadData)`

**Входные данные:**
```typescript
interface UpdateLeadData {
  clientReason?: string  // причина для клиента (при REJECTED)
  internalReason?: string // внутренняя причина
}
```

**Логика:**
- Проверить, что заявка принадлежит психологу
- Обновить статус
- Если статус `REJECTED` и есть `clientReason` — отправить письмо клиенту
- Записать `statusChangedAt`

#### 3.5 `createComplaint(data: CreateComplaintInput)`

**Входные данные:**
```typescript
interface CreateComplaintInput {
  fromType: 'client' | 'psychologist'
  fromId: string      // clientId или psychologistId
  toType: 'client' | 'psychologist'
  toId: string
  reason: string
  description?: string
  leadId?: string
}
```

**Логика:**
- Создать запись в `Complaint`
- Если жалоба на клиента (`toType = 'client'`):
  - Увеличить `complaintCount` у клиента
  - Если `complaintCount >= 3` — установить `isShadowBanned = true`

### 4️⃣ API роуты (опционально, можно через Server Actions)

Если нужны отдельные API роуты:
- `POST /api/leads` — создание заявки
- `GET /api/leads` — список заявок психолога (с фильтрацией)
- `GET /api/leads/[id]` — детальная заявка
- `PATCH /api/leads/[id]` — обновление статуса
- `POST /api/complaints` — создание жалобы
